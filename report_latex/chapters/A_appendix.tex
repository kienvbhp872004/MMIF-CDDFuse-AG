\section{Khảo sát thiết kế Module A và Module B (light retrain)}
\label{app:design-exploration}

Trước khi chốt phương án \emph{Adaptive Gating} cho Module A và \emph{Saliency-guided Pixel} cho Module B trong CDDFuse-AG (Chương~\ref{ch:method}), đồ án đã khảo sát \emph{ba phương án khác cho mỗi module} bằng \emph{light retrain} (10--25 epoch, đóng băng Encoder/Decoder, chỉ fine-tune fusion layer + biến thể). Mục đích là sàng lọc nhanh các candidate trước khi cam kết tài nguyên cho huấn luyện đầy đủ.

\paragraph{Lưu ý phương pháp luận.}
Kết quả light retrain ở phụ lục này \emph{không phải đóng góp chính} của đồ án vì:
\begin{enumerate}\itemsep0pt
    \item Cấu hình huấn luyện khác paper (epoch ít hơn, Encoder/Decoder đóng băng).
    \item Một phát hiện thực nghiệm (Mục~\ref{sec:disc-key}) cho thấy kết quả light retrain có thể \emph{không generalize} sang huấn luyện đầy đủ.
\end{enumerate}
Tuy nhiên, các số liệu này đã định hướng cho việc chọn phương án cuối cùng, vì vậy được lưu lại để tham khảo.

\subsection{Module A --- 4 phương án Fusion Rule}
\label{app:moduleA-ablation}

Bốn phương án thử nghiệm thay phép cộng $f_I + f_V$:
\begin{itemize}\itemsep0pt
    \item \textbf{A.1 Sum} (baseline): phép cộng đơn giản như CDDFuse gốc.
    \item \textbf{A.2 Gated} (chọn): cổng softmax mềm $g \odot f_V + (1-g) \odot f_I$, 16K params.
    \item \textbf{A.3 CrossAttn}: cross-modality attention kiểu Restormer MDTA, 37K params.
    \item \textbf{A.4 ChannelMoE}: per-channel mixture of 3 experts, 5K params.
\end{itemize}

\subsection{Module B --- 4 phương án Loss Target}
\label{app:moduleB-ablation}

Bốn phương án thay quy tắc max-pixel trong $L_{\mathrm{int}}^{II}$:
\begin{itemize}\itemsep0pt
    \item \textbf{B.1 Max} (baseline): $\max(I_Y, I_{IR})$ như CDDFuse gốc.
    \item \textbf{B.2 Mean}: $0.5 \cdot (I_Y + I_{IR})$.
    \item \textbf{B.3 Saliency} (chọn): trọng số gradient $w = |\nabla I_Y| / (|\nabla I_Y| + |\nabla I_{IR}|)$.
    \item \textbf{B.4 Learnable}: small CNN dự đoán $w$ từ $[I_Y, I_{IR}, \nabla I_Y, \nabla I_{IR}]$.
\end{itemize}

\paragraph{Phát hiện về methodology.}
Bảng~\ref{tab:app-combined-light} cho thấy \emph{magnitude} hiệu ứng giảm $3\text{--}5\times$ khi chuyển từ light retrain sang huấn luyện đầy đủ. Cả gain (NABF, QSF) lẫn regression (QM) đều bị ``co lại''. Đây là một bài học methodology được thảo luận trong Mục~\ref{sec:disc-key}: light retrain phù hợp cho \emph{sàng lọc} candidates nhưng \emph{không đáng tin} cho việc estimate hiệu ứng cuối cùng. Vì vậy mọi claim của đồ án về CDDFuse-AG đều dựa trên kết quả paper-faithful (Chương~\ref{ch:results}), không phải light retrain.

\subsection{Thử nghiệm bổ sung --- Module C (Decomp Loss)}
\label{app:moduleC}

Một phương án thử nghiệm thêm: thay Pearson Correlation Coefficient (CC) trong $L_{\mathrm{decomp}}$ bằng \textbf{linear CKA} (Centered Kernel Alignment) \cite{kornblith2019cka} để bắt được dependence phi tuyến. Mô hình \texttt{Combined-Saliency-CKA} được huấn luyện đầy đủ 120 epoch và đánh giá tương tự CDDFuse-AG:

\paragraph{Kết luận Module C.}
CKA decomposition loss \emph{không cải thiện} so với Pearson CC trong huấn luyện đầy đủ. Có thể do: (1) Encoder MDTA attention đã sinh ra feature gần linear-separable, CC đủ; (2) cỡ mẫu mỗi batch $N = 8$ nhỏ, ước lượng CKA (Gram matrix $8 \times 8$) không ổn định; (3) bottleneck nằm ở \emph{cấu trúc công thức} $D^2 / (1+B)$ chứ không phải measure dependence. Đây là một negative result có giá trị --- gợi ý hướng future work: cần reformulate \emph{cấu trúc công thức} decomp loss thay vì chỉ thay measure.

\section{Reproduce instructions}
\label{app:reproduce}

\begin{verbatim}
# 1. Clone repo
git clone https://github.com/kienvbhp872004/Image-Fusion
cd Image-Fusion

# 2. Setup Python 3.8 venv
py -3.8 -m venv .venv38
.\.venv38\Scripts\Activate.ps1

# 3. Install deps
pip install -r models/MMIF-CDDFuse/requirements.txt
pip install torch==2.1.2+cu118 torchvision==0.16.2+cu118 \
    --index-url https://download.pytorch.org/whl/cu118

# 4. Preprocess
cd models/MMIF-CDDFuse
python dataprocessing_MIF.py

# 5. Train
python train_MIF.py --variant Combined-Gated-Saliency --amp

# 6. Evaluate
python evaluate_cddfuse.py --variant Combined-Gated-Saliency \
    --modal CT --ckpt models/CDDFuse-Combined-Gated-Saliency_MIF_*.pth \
    --harvard_root ../../data/reference --save_perimage
\end{verbatim}
